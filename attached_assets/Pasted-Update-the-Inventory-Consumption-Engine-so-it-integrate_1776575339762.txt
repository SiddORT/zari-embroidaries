SECTION 1 — LINK ORDER CONSUMPTION LOGS WITH INVENTORY CONSUMPTION TABLE

---

When consumption recorded inside:

Swatch Order

Style Order

system must automatically create corresponding record in:

material_consumption table

Fields mapping:

item_id → from order consumption log

inventory_id → linked via inventory_items table

consumption_type = Swatch OR Style

reference_id = Swatch ID OR Style ID

consumed_quantity → from order consumption entry

consumed_by → logged-in user

consumption_date → order consumption timestamp

---

SECTION 2 — UPDATE INVENTORY CURRENT STOCK

---

After order consumption recorded:

system must automatically:

inventory_items.current_stock -= consumed_quantity

Then recalculate:

available_stock

Formula:

available_stock =
current_stock
− style_reserved_qty
− swatch_reserved_qty

---

SECTION 3 — UPDATE RESERVATION QUANTITY

---

If reservation exists for:

same item_id

same reference_id

same reservation_type

system must:

reduce reservation quantity

If:

reservation fully consumed

update:

reservation status = Converted

If:

partially consumed

keep:

status = Active

with remaining reserved quantity

---

SECTION 4 — CREATE STOCK LEDGER ENTRY

---

When consumption recorded inside orders:

system must create ledger entry:

transaction_type = Consumption

reference_type = Swatch OR Style

reference_number = reference_id

out_quantity = consumed_quantity

balance_quantity = updated_stock

remarks = Auto generated from order consumption entry
