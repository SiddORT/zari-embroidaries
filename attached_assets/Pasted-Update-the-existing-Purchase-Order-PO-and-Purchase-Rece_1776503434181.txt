Inventory
Swatch Costing

Follow structure exactly as defined below.

## OBJECTIVE

Centralize procurement so all Purchase Orders and Purchase Receipts work from one shared system.

Existing Costing PO and PR must now support:

Inventory-based procurement
Swatch-based procurement
Style-based procurement
Manual procurement

---

## SECTION 1 — ADD REFERENCE TYPE SUPPORT

Modify existing Purchase Order table.

Add fields:

reference_type
reference_id

reference_type values:

Inventory
Swatch
Style
Manual

reference_id stores:

Swatch ID
Style ID
NULL (for Inventory or Manual PO)

Example:

PO created from costing:

reference_type = Swatch

PO created from style planning:

reference_type = Style

PO created from inventory:

reference_type = Inventory
